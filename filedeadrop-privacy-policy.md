# Privacy Policy

**filedeadrop** — Last updated: June 16, 2026

---

## Overview

filedeadrop is a one-time, end-to-end encrypted file sharing service. This policy explains what information is collected when you use filedeadrop.com, how it is used, how long it is retained, and your rights with respect to it.

The short version: your files are encrypted in your browser before upload, the encryption key never reaches any server, and the file is permanently deleted after the first access or 24 hours — whichever comes first. The only personal data collected is standard server request metadata (IP address and user agent), retained for 30 days for security and operational purposes.

## 2. What data is collected

### 2a. Data you provide

filedeadrop does not require an account, registration, or any personal information to use. No name, email address, or identifying information is ever collected.

When you upload a file, the following occurs entirely in your browser before anything is transmitted:

- A one-time AES-GCM-128 encryption key is generated by your browser using the Web Crypto API.
- Your file and its filename are encrypted client-side. The encryption key is never transmitted to any server.
- Only the encrypted file bytes are uploaded — directly to Amazon S3 via a presigned URL. The application server never receives your file's contents.
- The encryption key, encrypted filename, and file identifier are embedded in the share link in the URL fragment (the `#` portion). URL fragments are never transmitted by browsers in HTTP requests and therefore never appear in any server log.

### 2b. Data collected automatically

Every request to the filedeadrop API generates a structured access log entry in Amazon CloudWatch. Each entry contains:

| Field | Description |
|---|---|
| IP address | The originating IP of the request |
| User agent | Browser and operating system string |
| Request ID | An AWS-generated identifier, not linked to you |
| Route | The route template (e.g. `GET /view/{id}`) — not the resolved file ID |
| Status code | HTTP response code |
| Response length and latency | Request performance metrics |
| Error message | Populated only on failed requests |

**What is not logged:** the encryption key (it never leaves your browser), the resolved file identifier on successful requests, file contents, and filenames.

### 2c. What is not collected

- Cookies — none are set.
- Session data — there are no sessions.
- Analytics or tracking scripts — none are present on the site.
- Third-party fonts or external resources — all assets are self-hosted and served from filedeadrop's own infrastructure.

## 3. How data is used

Automatically collected request metadata (IP address and user agent) is used solely for:

- Security monitoring and abuse prevention
- Diagnosing operational errors
- Understanding aggregate traffic patterns

This data is not used for advertising, profiling, or sale to third parties under any circumstances.

---

## 4. Data retention

| Data type | Retention period |
|---|---|
| Encrypted file (S3) | Deleted on first access, or after 24 hours — whichever comes first |
| File record (DynamoDB) | Deleted atomically on first access |
| API access logs (CloudWatch) | 30 days |
| Lambda execution logs (CloudWatch) | 30 days |

After the file is deleted from S3 and DynamoDB, it cannot be recovered. There is no backup.

---

## 5. Data storage and regional residency

filedeadrop is hosted on Amazon Web Services (AWS). At upload time, you choose a storage region:

- **US uploads** are stored and processed in AWS us-west-2 (Oregon).
- **EU uploads** are stored and processed in AWS eu-central-1 (Frankfurt, Germany).

Server access logs are written to CloudWatch in the same region as the upload. EU users' IP addresses and request metadata are logged in Frankfurt and do not leave the EU region.

AWS is the sole infrastructure sub-processor. AWS processes data under its own terms of service and data processing agreements. For more information, see [aws.amazon.com/privacy](https://aws.amazon.com/privacy/).

## 6. Data sharing

No personal data is sold, rented, or shared with third parties for marketing or any commercial purpose.

Data may be disclosed if required by law, court order, or valid legal process directed at the operator. Given the 30-day log retention and the architecture described above, the data available to disclose is limited to IP addresses, user agents, and request timestamps within that window. File contents and encryption keys are never available to disclose — the service is technically incapable of accessing them.

---

## 7. Security

Files are encrypted with AES-GCM-128 before leaving your device. The encryption key exists only in the share link's URL fragment, which browsers do not transmit in HTTP requests. This means the key is never held by filedeadrop's servers, never appears in any log, and is unrecoverable without the share link. Anyone with the share link can access the file exactly once.

You are responsible for protecting the share link. Once a file has been accessed, it is permanently deleted and cannot be retrieved.

---

## 8. File size limit

Files are limited to 25 MB. This limit is enforced at the API layer before a presigned upload URL is issued.
